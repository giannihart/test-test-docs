## Overview

Devscribe is a documentation platform that helps teams create, manage, and publish technical documentation. This frontend application is built with Next.js 15 and provides an intuitive interface for writing and organizing documentation.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** - Version 18.0 or higher
- **npm** or **yarn** - Package manager for dependencies
- **Git** - Version control system

## Installation

<Steps>
<Step title="Clone the repository">
Clone the project to your local machine:

<CodeSnippet language="bash">
git clone [repository-url]
cd devscribe-frontend
</CodeSnippet>
</Step>

<Step title="Install dependencies">
Install all required packages:

<CodeSnippet language="bash">
npm install
</CodeSnippet>

<Callout type="info">
Installation may take a few minutes as the project includes rich text editing, code highlighting, and documentation rendering libraries.
</Callout>
</Step>

<Step title="Configure environment variables">
Create a `.env` file in the root directory with the required configuration:

<CodeSnippet language="bash">
{`NEXT_PUBLIC_DEVSCRIBE_BACKEND_URL=http://127.0.0.1:8000`}
</CodeSnippet>

<Callout type="warning">
For OAuth authentication (Google/GitHub) to work properly, you must use `http://127.0.0.1:8000` exactly as shown - not `localhost`. The OAuth providers are strict about the redirect URL format.
</Callout>
</Step>
</Steps>

## Troubleshooting

<AccordionGroup>
<Accordion title="OAuth Authentication Issues">
**Problem**: Google or GitHub OAuth doesn't work

**Solution**: Verify your `.env` file contains:
<CodeSnippet language="bash">
{`NEXT_PUBLIC_DEVSCRIBE_BACKEND_URL=http://127.0.0.1:8000`}
</CodeSnippet>

The URL must use `127.0.0.1` instead of `localhost` for OAuth redirects to function properly.
</Accordion>

<Accordion title="Build Errors After Pulling Changes">
**Problem**: Application fails to start after pulling new changes

**Solution**: Clean the build cache and reinstall dependencies:
<CodeSnippet language="bash">
npm run clean
npm install
npm run dev:clean
</CodeSnippet>
</Accordion>

<Accordion title="TypeScript Errors">
**Problem**: Type checking failures

**Solution**: Run type checking to identify issues:
<CodeSnippet language="bash">
npm run typecheck
</CodeSnippet>

Address any reported type errors before building for production.
</Accordion>

<Accordion title="Port Already in Use">
**Problem**: Port 3000 is already in use

**Solution**: Either stop the process using port 3000 or specify a different port:
<CodeSnippet language="bash">
PORT=3001 npm run dev
</CodeSnippet>
</Accordion>
</AccordionGroup>

## Technology Stack

The application is built with modern web technologies:

- **Framework**: Next.js 15 with App Router
- **Language**: TypeScript
- **Styling**: Tailwind CSS 4
- **UI Components**: Radix UI primitives
- **Editor**: TipTap with ProseMirror
- **Code Highlighting**: Shiki
- **Documentation**: MDX with custom components
- **State Management**: React hooks and context
- **Authentication**: Supabase SSR
